# LogisticNet
Логистическая транспортная сеть с нелинейной стоимостью ребер и учетом перегруза 

## Постановка задачи

### Дано: 
- Ориентированный граф $𝐺=(𝑉,𝐸)$, где $𝑉$ – множество складов, $𝐸$ – множество дорог между складами
- Множество товаров $𝐾$, где для каждого товара $𝑘∈𝐾$:
  - $𝑠_𝑘∈𝑉$ - пункт отправки товара $𝑘$
  - $𝑡_𝑘∈𝑉$ - пункт прибытия товара $𝑘$
  - $𝑑_𝑘∈ℝ_+$ - перевозимый объем товара $𝑘$
- Для каждого ребра $𝑒∈𝐸$:
  - $𝑐_𝑒∈ℝ_+$ - стоимость использования одним транспортным средством
- Для каждого узла $𝑣∈𝑉$:
  - $𝑓_𝑣∈ℝ_+$ - стоимость перегрузки единицы товара на складу $𝑣$
  - 𝑊_𝑣∈ℝ_+ - максимальное допустимое количество перегруза на складу $𝑣$
- Вместимость одного транспортного средства: $𝐶$

### Переменные:
Пусть для каждого товара $𝑘∈𝐾$ задано $Π_𝑘$ - множество всех возможных путей из $𝑠_𝑘$ в $𝑡_𝑘$.
- $𝑥_𝑝^𝑘∈ℝ_+$ - поток товара $𝑘$ по пути $𝑝∈Π_𝑘$
- $𝑦_𝑒  ∈ℤ_+$ - количество ТС, используемых на ребре $𝑒∈𝐸$

### Целевая функция:
 $$𝒎𝒊𝒏(\sum_{𝒆∈𝑬}𝒄_𝒆∗𝒚_𝒆 + \sum_{𝒌∈𝑲}\sum_{𝒑∈𝚷_𝒌}\sum_{𝒗∈𝒑, 𝒗≠𝒔_𝒌, 𝒗≠𝒕_𝒌}𝒇_𝒗∗𝒙_𝒑^𝒌$$

### Ограничения
- Удовлетворение спроса для каждого товара:
  
$$\sum_{𝒑∈𝜫_𝒌}𝒙_𝒑^𝒌 = 𝒅_𝒌,  ∀𝒌∈𝑲$$
  
- Связь между потоком и количеством ТС:

$$\sum_{𝒌∈𝑲}\sum_{𝒑∈𝚷_𝒌:𝒆∈𝒑}𝒙_𝒑^𝒌 ≤ 𝑪∗𝒚_𝒆,  ∀𝒆∈𝑬$$
 
- Максимальный перегруз на складе:
  
$$\sum_{𝒌∈𝑲}\sum_{𝒑∈𝚷_𝒌}\sum_{𝒗∈𝒑, 𝒗≠𝒔_𝒌, 𝒗≠𝒕_𝒌}𝒙_𝒑^𝒌 ≤ 𝑾_𝒗,  ∀𝒗∈𝑽$$

## Реализованный подход решения задачи
В качестве основы для разработанного алгоритма была использована метаэквритсики General Variable Neighborhood Search (GVNS), окрестности которой имеют следующий вид:

1. Окрестность кратчайших путей с учётом перегруза
Для каждого товара генерируются новые кратчайшие пути в вспомогательном графе, где вес ребра включает не только стоимость проезда, но и стоимость перегрузки на складах. Позволяет находить экономически выгодные маршруты с минимальными транзитными затратами.
2. Окрестность частично загруженных ребер
Строится вспомогательный граф, в котором веса ребер учитывают остаточную вместимость транспортных средств (остаток от деления потока на вместимость ТС). Предпочтение отдаётся ребрам с частично заполненными машинами, что позволяет эффективно использовать свободное место и сокращать общее количество транспортных средств.
3. Окрестность обхода перегруженных вершин
Выявляются наиболее загруженные склады (по отношению текущего перегруза к максимальной пропускной способности). Для каждого товара генерируются новые пути, избегающие этих «узких мест» (путём временного удаления перегруженных вершин из графа). Позволяет перераспределять потоки на менее загруженные склады.
Так как со временем в наборе путей может набираться много неиспользованных путей, то после 2 и 3 окрестностей происходит очистка решения от путей с 0 потоком.

## Реализованный полход к генерации данных

Для тестирования алгоритма разработан генератор реалистичных логистических сетей. Население городов генерируется по закону Ципфа, после чего города размещаются на плоскости с помощью кластеризации, имитируя реальное пространственное распределение. Близлежащие города агрегируются в склады. Топология графа строится по модели Gastner–Newman с помощью Simulated Annealing, минимизируя компромисс между длиной сети и эффективным расстоянием между вершинами. Пропускная способность складов определяется с учётом населения и betweenness centrality, а матрица запросов на перевозки формируется с помощью doubly constrained гравитационной модели.
